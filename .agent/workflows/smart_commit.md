# Smart Commit Workflow

이 워크플로우는 코드를 커밋하기 전에 반드시 실행해야 합니다.
다음 5가지 항목을 점검하고, 그 결과를 사용자가 볼 수 있도록 요약하여 보고한 뒤 커밋을 진행하세요.

## 1. 5대 점검 항목

### 1) 스코프 준수 (Scope Compliance)
- 이번 PR/작업 범위를 벗어난 구현이 있는가?
- 불필요한 로직이나 오버 엔지니어링, 미래 기능을 미리 넣지 않았는가?

### 2) 변경 단위 (Atomicity)
- 이번 변경이 하나의 의미 있는 commit으로 묶였는가?
- 커밋 메시지가 작업 목적을 명확히 설명하는가? (Conventional Commits 준수)

### 3) 위험 요소 (Risk Factors)
- 권한, 인증, 데이터 저장, 외부 API 호출, 환경 변수 등 민감한 부분을 건드리지 않았는가?
- 보안상 위험한 코드는 없는가?

### 4) 다음 PR 영향 (Future Impact)
- 다음 작업에서 문제가 될 수 있는 임시 코드(TODO)나 기술 부채가 있는가?
- 있다면 명시하고 이슈화하거나 주석으로 남겼는가?

### 5) PM 관점 요약 (PM Summary)
- 이 PR(커밋)이 없으면 사용자 입장에서 어떤 문제가 발생하는지 한 문장으로 설명하라.
- 기술적인 내용보다는 '가치' 중심으로 서술하라.

## 2. 실행 절차

1. 위 5가지 항목에 대해 현재 변경 사항(`git diff` or `status`)을 분석합니다.
2. 분석 결과를 사용자에게 간략히 보고합니다 (필요시 `notify_user` 또는 대화 메시지).
3. 문제가 없다면 `git commit`을 진행합니다.
4. 커밋 메시지에 PM 관점 요약의 핵심 내용이 반영되면 더욱 좋습니다.
